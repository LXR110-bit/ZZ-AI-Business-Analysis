# model-tag-monitor v1.1.0 release checklist

## Version
- Baseline: v1.0.0
- Release: v1.1.0

## Required production env
PM2 must be restarted with `model-tag-monitor/ecosystem.config.js` so the running Node process has:
- `IMPORT_DIR=/root/workspace/ZZ-AI-Business-Analysis-base-migration/data/imports`
- `KEEP_WEEKS=10`
- `TARGET_WEEKS=2026-W18,...,2026-W27`
- `DASHBOARD_URL=http://47.84.94.234:8848/?tab=dashboard`

## Test before merge
1. Local tests: `cd model-tag-monitor && npm test`
2. Local real-data sync smoke:
   `IMPORT_DIR=/private/tmp/zz_ai_imports_20260707/data/imports TARGET_WEEKS=2026-W18,2026-W19,2026-W20,2026-W21,2026-W22,2026-W23,2026-W24,2026-W25,2026-W26,2026-W27 node -e "require('./src/sync').sync().then(console.log)"`
3. API contract smoke: `/api/dashboard` returns `version=1.1.0`, `week=2026-W27`, `weeks=W18..W27`, and non-empty `board/tiers/categories`.

## Server rollout order
1. Deploy git release branch to server.
2. `npm ci --omit=dev` under `/root/model-tag-monitor` if dependencies changed.
3. `pm2 startOrReload ecosystem.config.js --update-env`.
4. Run `/root/model-tag-monitor/scripts/refresh-dashboard-daily.sh` once with `FEISHU_DRY_RUN=1`.
5. Verify server data and online API both show W18-W27.
6. Only after server run succeeds, merge to main.
7. Run the real online flow once without dry-run to validate the Feishu style-2 card.
