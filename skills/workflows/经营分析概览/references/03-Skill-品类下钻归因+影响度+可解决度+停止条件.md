# Skill 3：品类下钻归因 + 影响度 + 可解决度 + 停止条件

> **版本**：v0.4 | **依赖**：Skill 2 标记为"触发品类下钻"的品类
> **输入**：异常品类下各品类的 估价UV/下单UV/成交订单/GMV/客单价 环比 + 品类归属的簇名
> **输出**：拖累/带涨品类 + 流量/转化拆分 + 影响度% + 可解决度 + 建议操作
> **必答老板问题**：Q3 哪里该加注/收手？+ 这个异常值不值得救？

---

## 执行步骤

### 第1步：定位核心异常品类（定量计算）

```
对品类簇下每个品类，计算：
  异常得分 = |估价UV环比| + |成交订单环比| + 连续周数权重（连续2周=+5，连续3周=+10）
```

简单版：得分最高的 **1-2 个品类**标记为「核心异常品类」。

**阈值的三个参考线**：

| | 低风险 | 需关注 | 异常 |
|---|---|---|---|
| 估价UV环比 | -2% ~ +2% | -2% ~ -5% 或 +2% ~ +5% | < -5% 或 > +5% |
| 成交订单环比 | -2% ~ +2% | -2% ~ -5% | < -5% |
| 连续周数 | 1 周 | 连续 2 周 | 连续 3 周 |

---

### 第2步：流量/转化分开归因（与Skill 2 方法一致，粒度更深）

```
对每个核心异常品类：
  流量异常：估价UV 环比 < -5% → 问题在「流量端」
  转化异常：成交订单环比 - 估价UV环比 < -3pp → 问题在「转化端」
  两者都有 → 「全线承压」
  注意：同品类内，流量端在跌、转化端在涨可以并存，必须分开说
```

**老板痕迹**：W27 摄影摄像「流量-6.8%、转化+0.31pp」= 流量端承压、转化端修复。

---

### 第2.5步：品类间此消彼长判断（v0.4 新增，防止归因错位）

**第一性原理**：平台总流量有限。一个品类跌，不一定是"外部行情"，可能是流量被同簇另一个品类蚕食了（内部竞争）。把内部竞争误判为外部行情，会导致完全相反的对策。

```
对每个下跌品类：
  1. 该品类流失的估价UV，同期同簇是否有品类在涨？
  2. 对比同簇各品类的估价UV排名变化（本周 vs 上周）

判断：
  ├─ 跌的品类流量 ≈ 涨的品类增量，且总量基本不变
  │   → 【内部竞争】流量在簇内转移，不是外部行情问题
  │   → 对策：调整品类间的流量分配/曝光位，而非救单个品类
  │
  ├─ 跌的品类没有对应的涨，整簇总量在缩
  │   → 【外部流失】真的是外部行情/需求下降
  │   → 对策：按原框架归因（行情/动作/竞对）
  │
  └─ 跌的品类流量流向了"其他簇"
      → 【跨簇转移】用户需求迁移，需上报 Skill 5 做结构性判断
```

**示例**：若显卡估价UV -16% 而同簇内存条估价UV +14%，总量持平 → 内部竞争（用户预算在DIY配件间转移），不该把显卡当"行情走弱"救，而应看整簇DIY是否健康。

**输出**：`此消彼长：[内部竞争(流向X品类) / 外部流失 / 跨簇转移(流向Y簇)]`

---

### 第3步：影响度计算（必做，决定值不值得继续投入）

```
影响度% = 异常品类 GMV 波动量 / 所属簇 GMV 总量 × 100%
```

| 影响度 | 优先级 | 老板反应 |
|---|---|---|
| >20% | **P0 必须解决** | 直接追问，调资源 |
| 10-20% | P1 优先解决 | 要求下钻，给方案 |
| 5-10% | P2 可暂缓 | 标注，但不深挖 |
| <5% | P3 忽略 | 不上会提 |

**示例**：
- W27 运动相机「GMV-12.0% 是最大单点拖累」→ 影响度算出来必然>10%
- 内存条「订单-4.3% 但客单价+25.4%」→ 影响度低，P3

---

### 第4步：可解决度判断 + 建议操作（停止条件）

```
问：这个异常，是"我们自己的问题"还是"市场行情"？
```

| 元凶 | 可解决度 | 建议操作 | 停止条件 |
|---|---|---|---|
| 定价问题（我们提价/竞品压价） | 高 | **调整定价，下周验证** | 转化率恢复 或 定价调整无效→改变策略 |
| 流量问题（渠道/活动/曝光下降） | 中 | **检查流量入口，补充渠道** | 流量恢复 或 渠道不可复现→改方向 |
| 行情问题（大盘/行业下跌） | 低 | **暂缓观察，等行情恢复** | 行情恢复 或 持续下跌 3 周→放弃 |
| 品类结构问题（高价/低价品类占比变） | 中 | **调整品类策略** | 结构恢复 或 结构持续恶化→做产品决策 |
| 竞争对手动作（友商顶价/降价抢量） | 低 | **持续监控，评估是否跟进** | 跟进动作起效 或 放弃跟进 |
| 不可解（季节/政策/用户习惯） | 不可解 | **停止投入，资源转到其他品类** | 无需等待 |

**每一步操作后，老板必须能回答"我怎么知道这事有眉目了？"** 这就是停止条件。

---

### 第5步：是否值得继续下钻到机型层

满足以下任一条件 → `值得下钻`：
- 影响度 > 20%（P0）
- 环比 >±5% 且连续 2 周同向
- 品类是老板关注重点品类（组装机/运动相机/无人机/白酒/显卡/内存条/台球杆）

**不满足 → 停止，输出品类层结论，不继续机型下钻。**

---

### 风险等级（品类粒度）

| 风险等级 | 触发条件 |
|---|---|
| 🔴 危险 | 影响度 >20% 且连续 3 周下跌 / 履约驳回率 >30% |
| 🟡 警告 | 影响度 10-20% 或 环比连续 2 周同向异常 |
| 🟢 正常 | 影响度 <10% 且波动在容忍范围 |

---

## 输出格式

```
【品类下钻】异常品类：XXX | 所属簇：XXX | 风险等级：🔴/🟡/🟢

异常定位：
  - 异常得分：[分值]（估价UV_% + 成交订单_% + 连续权重_）
  - 影响度：[__%]（P0/P1/P2/P3）
  - 连续趋势：[上升/下降/波动] 连续 [N] 周

流量端：估价UV __（环比__%）→ [正常/异常：原因推测]
转化端：成交订单 __（环比__%）vs 估价UV __% → 差值 __pp → [修复/恶化/平稳]
此消彼长：[内部竞争(流向X品类)/外部流失/跨簇转移(流向Y簇)]

可解决度：[高/中/低/不可解]
  元凶：[定价问题/流量问题/行情问题/品类结构/竞争对手/不可解]
建议操作：[具体操作]
停止条件：[什么信号意味着动作起效 或 应该放弃]

值得下钻到机型层：是/否（依据：[影响度>20%/连续2周/重点品类]）
  - 若是，下钻方向：[定价/流量/转化]
```

## 老板原文对照

> 组装机订单-6.6%、显卡-16.3%、台式机-22.6% → 三品类同时受累，定位为簇层异常
> 运动相机 GMV-12.0% 是最大单点拖累 → 影响度高、P0
> 内存条订单-4.3% 但客单价+25.4%（少数大单驱动，不是量的增长）→ 影响度低、P3、不用下钻
