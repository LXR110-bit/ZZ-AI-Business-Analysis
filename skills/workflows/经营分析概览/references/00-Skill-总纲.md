# 老板分析框架 · 总纲

> 版本：v0.4 | 更新：2026-07
> 用途：放在飞书目录根节点，说明这套框架的使用方式和整体逻辑。每次写周报/分析报告前先读这一篇。

---

## 〇、两条链路：分析链 vs 表达链（金字塔原理）

**这套框架有两条方向相反的链路，不能混为一谈：**

| | 分析链（自下而上） | 表达链（自上而下） |
|---|---|---|
| 方向 | 数据 → 归因 → 结论 | 结论 → 论据 → 数据 |
| 执行 | Skill 1 → 2 → 3 → 4 | Skill 5 输出时倒过来 |
| 目的 | 保证结论**推得出来** | 保证老板**读得高效** |
| 依据 | A–E 五大判断框架 | 金字塔原理 |

**金字塔原理的三条铁律，落在本框架的位置：**

1. **结论先行**：Skill 5 输出必须塔尖在最上（单一核心判断 + 动词），不能让老板从 Skill 1 读到 Skill 5 才知道结论
2. **以上统下**：每条核心发现必须挂 `[来源 Skill + 数据]`，老板问"凭什么"能立刻追溯
3. **MECE 分组**：核心发现按老板三问（Q1安全/Q2兑现/Q3资源）分组，不重叠、不遗漏

```
       序言（SCQA）
          ↓
    塔尖：单一核心判断 + 动词
      ↓      ↓      ↓
   Q1安全  Q2兑现  Q3资源     ← MECE 三组
    ↓       ↓       ↓
  Skill证据 Skill证据 Skill证据  ← 可追溯
          ↓
     Q4：下周行动
```

## 一、核心认知：老板读周报的"四问驱动"

老板看任何数据，脑子里只有四个问题。**任何 Skill 的输出，必须回答其中至少一个，否则就是无效分析。**

| 老板的问题 | 含义 | 由哪个 Skill 回答 |
|---|---|---|
| **Q1 大盘安全吗？** | 有没有系统风险？有没有外部冲击？ | Skill 1 |
| **Q2 上周判断对了吗？** | 上周的预判/策略，这周兑现了多少？ | Skill 2（策略验证）+ Skill 5（兑现率） |
| **Q3 哪里该加注/收手？** | 资源怎么重新分配？ | Skill 2 + Skill 3 |
| **Q4 今天/这周做什么？** | 必须有明确行动指向，不能含糊 | Skill 5 |

**框架的硬约束**：每个 Skill 的输出，开头必须明示回答了上面哪几个问题。

---

## 二、五大判断框架（A–E）

| 框架 | 老板用它回答什么 | 执行位置 |
|---|---|---|
| **A 三层穿透** | 上游变了吗？中间市场变了吗？我们自己变了吗？ | 归因时必须检查，不能只盯着自己 |
| **B 生命周期×阈值** | 这个品类/渠道现在在什么阶段？涨是好事还是回光返照？ | 每个判断句必须带阶段标签 |
| **C 价值链卡点** | 当前链条上哪个环节是瓶颈？优化非瓶颈没用 | 下钻到品类/机型层时必用 |
| **D 异动诊断四问** | 信号是真的吗？短期还是结构性？我们还是市场？能对应到动作吗？ | 发现异常信号第一反应 |
| **E 闭环证据链** | 我做的动作有没有用？基线→增量→成本→ROI 闭环了没？ | 结论必须带"下一步" |

## 三、执行链路（5 个 Skill 的调用顺序）

```
Skill 0：总纲（你正在读）
  │
Skill 1：数据可信度 + 三基准对比 + 大盘链路定性 + 风险等级 + 上周预判检核
  ├─ 输入：全链路 本周/上周/4周前/近8周序列
  ├─ 输出：数据可信度 + 三基准(环比/同比/近8周) + 趋势标签 + 定性命名 + 链路传导 + 风险等级🔴🟡🟢 + 上周预判兑现
  └─ 触发条件：每周必跑（第一个）；Step -1 数据不可信则全报告降级"待核"
  │   → 异常簇 / 异常环节 → 推送给 Skill 2
  │
Skill 2：品类簇归因 + 策略验证 + 资源分配信号
  ├─ 输入：发展/孵化/种子 各簇数据 + 上周策略清单
  ├─ 输出：流量/转化分开归因 + 行情vs动作 + 策略验证 + 资源分配建议
  └─ 触发条件：每周必跑
  │   → 异常品类 → 推送给 Skill 3
  │
Skill 3：品类下钻 + 影响度 + 可解决度
  ├─ 输入：异常品类的估价UV/下单UV/成交订单/GMV
  ├─ 输出：拖累/带涨品类 + 流量/转化拆分 + 影响度% + 可解决度 + 建议操作
  └─ 触发条件：Skill 2 标记异常的品类才跑
  │   → "值得下钻" → 推送给 Skill 4
  │
Skill 4：机型规律提炼 + 品类扩展验证（按需）
  ├─ 输入：异常品类下各机型数据
  ├─ 输出：问题/机会机型 + 机型规律 + 跨品类复用判断
  └─ 触发条件：仅 Skill 3 输出"值得下钻=是"时跑
  │
Skill 5：综合判断 + 情绪基调 + 下周行动
  ├─ 输入：Skill 1–4 全部输出
  ├─ 输出：一句话压缩 + 整体风险等级 + 兑现率 + 下周最优先行动
  └─ 触发条件：必须最后跑
```

### 四个硬约束

1. **数据可信度先于一切**（v0.4）——Skill 1 的 Step -1 若判定数据存疑，整份报告结论加"待核"标签，风险等级不下🟢。数据不准时，漂亮的结论比没有结论更危险。
2. **Skill 1 必须第一个跑**——它给整份报告定调，包括风险等级
3. **Skill 4 只在 Skill 3 判断"值得下钻"时触发**——避免无意义下钻浪费时间
4. **每个 Skill 输出必须带风险等级 🔴🟡🟢**——老板第一眼要先知道"该多紧张"

---

## 四、风险等级定义（所有 Skill 通用）

| 等级 | 含义 | 触发条件 | 老板的反应 |
|---|---|---|---|
| 🔴 危险 | 系统风险，必须当天解决 | 大盘 GMV 环比 <-3% / 核心品类连续 3 周下跌 / 履约超时率 >20% | 直接追问、调资源、跨部门拉会 |
| 🟡 警告 | 局部风险，有恶化趋势 | 单簇/单品类环比 <-5% / 转化率连续 2 周下滑 / 上周预判未兑现 | 要求下钻、给解决方向 |
| 🟢 正常 | 链路健康，波动在容忍范围 | 全链路环比 <-2% 且无结构性异常 | 跳过，不深挖 |

**优先级规则**：整份报告的整体风险等级 = 各 Skill 输出的最高风险等级。

---

## 五、上周预判检核（所有 Skill 必做）

第一性原理：**老板要新周报，是要评估"上周判断对不对"。**

每个 Skill 输出时，必须引用上一期同 Skill 的结论做对比：

```
【上周预判】XXX
【本周兑现】[兑现/部分兑现/未兑现/反向]
【偏差原因】[若未兑现，是判断错了还是外部变了]
```

**没有这一项，老板会自己翻上周周报来对——那就是框架没覆盖他的工作流。**

---

## 六、资源分配信号（Skill 2/3/5 必做）

第一性原理：**老板看完数据，下一句一定是"所以钱/人/流量投到哪？"**

每个异常点必须同时给：

| 维度 | 含义 | 取值 |
|---|---|---|
| **影响度** | 这个异常影响大盘多少 GMV/订单？ | 影响大盘 GMV 比例 <0.5% / 0.5-2% / >2% |
| **可解决度** | 几周内能改变吗？ | 等市场恢复 / 自己可控可改 / 不可改 |
| **建议操作** | 该怎么应对？ | 继续投入 / 暂缓观察 / 放弃 / 下钻 |

---

## 七、老板的"一句话压缩语法"（所有 Skill 输出规范）

**v0.3 起，塔尖是两段式，不是四成分平铺。**

```
第一句 核心判断：[整体定调] +（风险等级）+ [动词：加注/持平/收手 + 对象]
第二句 量价拆解：[GMV 拆解] + [真假增长定性]
```

动词必须三选一且指向具体对象：**加注**（上行且可控）/ **持平**（无异常）/ **收手**（策略失效或下行）。

**示例（塔尖）**：
- 核心判断：本周大盘承压（🟡），对运动相机收手、对显卡加注。
  量价拆解：GMV -2.0%（订单 -4.2% × 客单价 +2.4%），量退价补。
- 核心判断：本周首次突破 5000 单（🟢），对整箱回收持续加注。
  量价拆解：订单 +18.7%，真增长，进入规模化阶段。

**四成分仍是内部素材来源**（供第一句合成），但不再直接平铺给老板：
`[定性名字]`(Skill1) + `[量价拆解]`(Skill1) + `[主动/被动标签]`(Skill2) + `[阶段判断]`(Skill5)

## 八、阶段标签库（Skill 5 必用）

| 阶段标签 | 使用场景 |
|---|---|
| `进入规模化阶段` | 订单首次突破整数关口 |
| `价格硬刚占领心智的阶段` | 竞品在顶价，必须跟进 |
| `不进则退` | 竞品已建立价格优势 |
| `行情消化后流量回落风险` | 新品效应带来流量但不可持续 |
| `远程质检是核心竞争力` | 白酒/名酒等多单率品类 |
| `转化端修复中，流量端承压` | 流量跌但转化涨 |
| `少数大单驱动，非量的增长` | 客单价拉起的 GMV |
| `下行通道` | 大盘流量持续走弱 |
| `上行通道` | 大盘流量持续走强 |

## 九、数据输入规范

- 环比精确到小数点后 1 位；pp 变化精确到小数点后 2 位
- **三基准（v0.4，Skill 1 必给）**：环比(vs上周) + 同比(vs4周前) + 近8周均值对比。单看环比会被季节性误导
- **输入需含近 8 周序列**（不只本周+上周），否则无法做趋势标签和"近N周首次"判断
- 判断阈值：环比 >±5% 或连续 3 周同向变化 → 标记异常
- "少数大单驱动"判定：客单价涨但订单跌，且客单价涨幅 > GMV 跌幅的 2 倍
- **数据可信度（v0.4）**：三查（同步时点/口径断层/比值异常）任一不过 → 标"存疑"，结论降级"待核"

### 归因规范（v0.4，所有 Skill 通用）

归因不再用单标签，一律走二维矩阵，每维各带证据：
```
归因 = [责任主体] × [时间属性]
责任主体（MECE，可多个各带证据）：市场行情 / 我方动作 / 竞对动作 / 用户结构变化 / 不可归因
时间属性（MECE，三选一）：一次性事件 / 短期波动 / 结构性趋势
```
「不可归因」不是逃生舱：必须说明查了哪些主体、都排除了，才能用。
