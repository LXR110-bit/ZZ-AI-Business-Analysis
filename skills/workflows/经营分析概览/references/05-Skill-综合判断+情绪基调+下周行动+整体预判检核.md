# Skill 5：综合判断 + 情绪基调 + 下周行动 + 整体预判检核

> **版本**：v0.3（金字塔重构） | **依赖**：Skill 1–4 的全部输出
> **输入**：Skill 1 的定性命名 + Skill 2 的策略验证 + Skill 3 的影响度 + Skill 4 的机型规律（若有）
> **输出**：金字塔式周报——序言(SCQA) + 塔尖(单一判断) + 关键论据(三问分组) + 下周行动
> **必答老板问题**：Q1 安全吗？+ Q2 上周判断对了吗？+ Q3 资源怎么动？+ Q4 今天做什么？

---

## 金字塔原理如何贯穿本 Skill

**核心认知**：Skill 1–4 是「分析链」（自下而上推理），Skill 5 是「表达层」（自上而下呈现）。老板打开周报，视线路径必须是：

```
       序言（SCQA：一句话交代来龙去脉）
              ↓
       塔尖（单一核心判断 + 动词：加注/持平/收手）
         ↓        ↓        ↓
      Q1安全？  Q2兑现？  Q3资源怎么动？   ← 关键论据按老板三问 MECE 分组
       ↓         ↓          ↓
    [Skill证据] [Skill证据] [Skill证据]     ← 每条论据挂数据凭据，可追溯
              ↓
       下周行动（Q4，1-2条带停止条件）
```

**硬约束**：
1. 塔尖必须是**单一判断句**，不能平铺四个成分让老板自己拼
2. 每条核心发现必须标注 `[来源 Skill + 数据]`，老板问"凭什么"时能立刻追溯
3. 归组必须 MECE：Q1/Q2/Q3 三组不重叠、不遗漏

---

## 执行步骤

### 第1步：写序言（SCQA，2-3句话）

老板每篇周报开头都有一个隐藏的来龙去脉，必须显性提炼：

```
S 情境（Situation）：上周处于什么状态？（从上周预判/上周定性拿）
C 冲突（Complication）：本周发生了什么打破了那个状态？（从 Skill 1 定性拿）
Q 疑问（Question）：所以现在最该回答什么？（自然导出）
A 回答（Answer）：塔尖的核心判断（下一步给出）
```

**示例**：
> 上周判断大盘随月度趋势进入下行通道（S），本周 GMV 近 8 周首次负增长、呈量退价补（C），是继续下行还是已见底？（Q）

---

### 第2步：写塔尖（单一核心判断 + 动词）

**塔尖 = 两句话，不是四个成分平铺。**

```
第一句 核心判断：[整体定调] + [风险等级] + [动词：加注/持平/收手 + 对象]
第二句 量价拆解：[GMV 拆解] + [真假增长定性]
第三行 情绪基调：[🚀兴奋 / 🙂稳健 / 😐警惕 / 😰紧张]
```

**动词库（必须三选一，且指向具体对象）**：
| 动词 | 触发条件 | 示例 |
|---|---|---|
| 加注 | 有品类/机型走上行通道且可控 | 对显卡加注 |
| 持平 | 无明显异常，维持现状 | 大盘持平观察 |
| 收手 | 有品类主动策略失效/趋势下行 | 对运动相机收手 |

**示例**：
> 核心判断：本周大盘承压（🟡），应对运动相机收手、对显卡加注。
> 量价拆解：GMV -2.0% = 订单 -4.2% × 客单价 +2.4%，量退价补。
> 情绪基调：😐 警惕

**情绪基调映射（MECE）**：
| 情绪基调 | 触发 |
|---|---|
| 🚀 兴奋 | 整体🟢 + 核心指标突破历史 |
| 🙂 稳健 | 整体🟢 无异常 |
| 😐 警惕 | 整体🟡 局部异常 |
| 😰 紧张 | 整体🔴 系统风险 |

---

### 第3步：写关键论据（按老板三问 MECE 分组）

**不按 Skill 编号排，按老板脑子里的三个问题分组。** 每条发现必须挂 `[来源 + 数据]`。

```
Q1 大盘安全？（安全性论据，来自 Skill 1）
  • [发现 + 关键数据]（Skill 1）
  • [链路瓶颈/逆势环节]（Skill 1）

Q2 上周预判兑现？（有效性论据，来自 Skill 2 策略验证）
  • [策略X → 兑现/未兑现/反向 + 数据]（Skill 2）

Q3 资源怎么动？（资源论据，来自 Skill 2/3/4）
  • [某品类收手/加注 + 影响度 + 可解决度]（Skill 3）
  • [机型规律/跨品类复用]（Skill 4，若有）
```

**MECE 校验清单**（输出前自检）：
- [ ] Q1 只放"大盘是否安全"的证据，不混入资源建议
- [ ] Q2 只放"上周判断对不对"，不混入本周新发现
- [ ] Q3 只放"钱/流量往哪投"，不混入风险描述
- [ ] 每条发现有且只属于一组（不重叠）
- [ ] 三组覆盖了 Skill 1–4 的所有关键输出（不遗漏）

---

### 第4步：上周预判兑现率总览

```
兑现率 = 兑现数 / 总预判数

- 总预判数：N 条
- 兑现：M 条（M/N %）
- 未兑现：K 条 + 原因
- 反向兑现：L 条 + 原因

兑现率打分：
  ├─ ≥ 70%   → 预判质量高
  ├─ 40-70%  → 有偏差，下周调整判断依据
  └─ < 40%   → 外部超预期，复盘判断框架
```

---

### 第5步：下周最优先行动（Q4，1-2条带停止条件）

**只给 1-2 条。每条必须带停止条件（金字塔的"可验证"要求）。**

```
行动格式：
  [具体行动] + [为什么（挂 Skill 凭据）] + [停止条件（什么信号=起效/该放弃）]
```

**示例**：
> 1. 恢复运动相机价格竞争力，观察 2 周（Skill 2：放弃价格后转化跌幅>流量降幅）。停止条件：转化率回升至基线±1pp 内→保留；2 周不回升→问题不在价格，转 Skill 4 机型下钻。
