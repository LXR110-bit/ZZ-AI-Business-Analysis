# Skill 2：品类簇归因 + 策略验证 + 资源分配信号

> **版本**：v0.4 | **依赖**：Skill 1 的输出（异常簇列表+整体风险等级+定性标签）
> **输入**：发展/孵化/种子 三簇的 估价UV/下单UV/成交订单/GMV/客单价 环比 + 上周策略清单
> **输出**：每簇流量/转化分开归因 + 行情vs动作 + 策略验证 + 资源分配建议
> **必答老板问题**：Q2 上周判断对了吗？+ Q3 哪里该加注/收手？

---

## 执行步骤

### 第0步：上周策略验证（必须先做）

```
问：上周针对每簇做了什么策略/动作？本周数据是否验证了策略有效？
```

对每簇检查：

| 上周策略类型 | 本周应该看到的数据信号 | 若没看到 = 策略失效 |
|---|---|---|
| 投流量（push/资源位） | 估价UV涨幅 > 簇平均 | 流量没起来 → 策略失效 |
| 调定价/提价 | 客单价涨 + 转化率稳 | 客单价涨但转化跌 > 1pp → 提价过度 |
| 放缓价格竞争力（W27运动相机） | 转化率跌幅 vs 流量跌幅 | 转化跌幅 > 流量降幅 → 放弃价格得不偿失 |
| 新品发布承接（W26 Pocket 4P） | 估价UV + 成交订单同步涨 | 只涨流量不涨订单 → 承接页/定价有问题 |
| 模板优化（W26拍照模板） | 履约时长下降 + 完结率提升 | 时长没降 → 模板方向错 |

**输出时必须给**：每簇至少一条「上周策略→本周兑现情况」判断。

**老板痕迹**：
- W27「运动相机放缓价格竞争力」→ 验证结论：「随流量快速下跌，放弃价格优势是有所利空的，友商还在顶价」= 策略反向兑现
- W26「拍照模板优化」→ 验证结论：「只能做减法、不能做加法，数码相机增加精准标准反而全面恶化」

---

### 第1步：流量端 vs 转化端分开判断（不能混在一起）

对每个簇分别计算：

```
流量端信号：估价UV 环比
转化端信号：成交订单环比 - 估价UV环比
  - 差值 > 0 → 转化在改善
  - 差值 < -3pp → 转化在恶化
  - 差值在 [-3, 0] → 转化基本平稳
```

| 流量 | 转化 | 判断 |
|---|---|---|
| 跌 | 涨 | 「转化端修复，流量端承压」 |
| 涨 | 跌 | 「流量端拉动，转化端恶化」 |
| 涨 | 涨 | 整体好 |
| 跌 | 跌 | 整体差 |

**老板痕迹**：
- W27 摄影摄像：「流量端承压最明显，转化端修复」
- W27 孵化品类：「流量平稳，转化端提升」

### 第2步：归因二维矩阵（v0.4 升级，责任主体 × 时间属性）

**旧版"行情/动作/双重/待查"不 MECE**（双重=重叠，待查=逃生舱）。v0.4 拆成两个正交维度，分别判定，各带证据。

**维度一：责任主体（MECE 五选一，回答"谁造成的"）**
| 责任主体 | 检查方式 | 证据要求 |
|---|---|---|
| 市场行情 | 有公开行情信号（显卡涨价/新品发布/以旧换新政策） | 同簇其他品类是否同步动？ |
| 我方动作 | 本周有运营动作（活动/补贴/push/提价） | 动作覆盖品类是否同步动？ |
| 竞对动作 | 友商顶价/降价/抢量 | 竞品价格/流量对比数据 |
| 用户结构变化 | 高/低客单用户占比变化 | 客单价结构、以旧换新比例 |
| 不可归因 | 三查后仍无信号（**必须真的查过，不是偷懒**） | 需说明查了哪些、都排除了 |

> 允许多个主体同时成立（如市场行情 + 我方动作），但每个成立的都要单独给证据。这与旧版"双重带动"的区别：旧版是一个模糊标签，新版是两个各自带证据的独立判定。

**维度二：时间属性（MECE 三选一，回答"会持续吗"）**
| 时间属性 | 特征 | 对策含义 |
|---|---|---|
| 一次性事件 | 大促/单笔大单/政策窗口，不可复现 | 别当趋势，回落是正常 |
| 短期波动 | 1-2周内可回归的正常抖动 | 观察，不动作 |
| 结构性趋势 | 连续3周同向 + 同比印证（对齐 Skill 1 趋势标签） | 必须干预或战略调整 |

**输出**：`归因 = [责任主体(可多个,各带证据)] × [时间属性]`
示例：`显卡涨 = 市场行情(同簇台式机同步涨) × 结构性趋势(连续3周+同比正)`

---

### 第2.5步：论点交叉仲裁（v0.4 新增，解决 Skill 1 与 Skill 2 打架）

**问题**：Skill 1 可能定性"量退价补"（被动、市场带），但 Skill 2 归因发现是"我方动作"（主动）——两个标签矛盾时谁说了算？

```
仲裁规则：
  1. 若 Skill 1 定性 = 被动(市场带)，但本簇归因主体 = 我方动作
     → 检查：这个我方动作是"全局"还是"局部"？
       ├─ 局部(仅此簇) → 保留 Skill 1 大盘被动定性，标注"本簇为主动逆势"
       └─ 全局(影响大盘) → 反馈给 Skill 5，修正塔尖标签为"主动"

  2. 若多簇归因主体不一致（发展=市场带，孵化=我方动作）
     → 塔尖标签取"影响度加权后的主导主体"，并在核心发现里分簇说明

  3. 冲突无法解决 → 标注"标签存疑"，交 Skill 5 时并列呈现，不强行统一
```

**输出**：若发生仲裁，必须在本 Skill 输出末尾标注 `【仲裁】原判X，因证据Y，修正为Z`。

---

### 第3步：资源分配信号（必做，老板下一句一定问）

对每个簇，给三元组：

| 维度 | 取值 | 老板的反应 |
|---|---|---|
| **影响度** | 影响大盘GMV比例 <0.5% / 0.5-2% / >2% | 影响度 <0.5% → 不值得花时间；>2% → 必须处理 |
| **可解决度** | 等市场恢复 / 自己可控可改 / 不可改 | 自己可控 → 立刻动作；等市场恢复 → 暂缓 |
| **建议操作** | 继续投入 / 暂缓观察 / 放弃 / 下钻 | 直接给操作 |

**示例**：
- 发展品类·电脑办公：「影响度2.1%，可解决度=自己可控（提价/调整品类结构），建议继续投入显卡+调整组装机定价」
- 摄影摄像·运动相机：「影响度1.8%，可解决度=自己可控（价格策略），建议下周恢复价格竞争力」

---

### 第4步：触发下钻判断

某簇满足以下任一条件 → 标记 `触发品类下钻`：
- 簇内某品类环比 >±5%
- 流量或转化连续 2 周同向下滑
- 影响度 >2% 且可解决度=自己可控

---

## 输出格式

```
【发展品类】（🟢/🟡/🔴）

上周策略验证：
  - 上周策略：[策略名]
  - 本周兑现：[兑现/部分兑现/未兑现/反向]
  - 偏差原因：[原因]

流量端：估价UV __（环比__%）→ [承压/平稳/拉动]
转化端：成交订单 __（环比__%）→ [修复/恶化/平稳]
归因：[责任主体(可多个,各带证据)] × [时间属性] + 证据：[具体品类或动作]
资源分配：影响度 __%，可解决度 [等市场恢复/可控/不可改]，建议 [继续投入/暂缓/下钻/放弃]
触发品类下钻：是/否（标记具体异常品类）
```

【仲裁】（若与 Skill 1 定性标签冲突时才输出）
  原判：[Skill 1 标签] → 因：[证据] → 修正为：[新标签 / 保留+局部说明 / 存疑并列]

孵化品类、种子品类同此格式。
