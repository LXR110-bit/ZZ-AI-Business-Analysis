"""
Step 3 spawn_agent prompt：品类归因 + 策略验证 + 品类下钻

输入：JSON，包含 Step 2 的链路判断结论 + 发展/孵化/种子三簇数据 + 上周策略清单
输出：严格 JSON，必须符合以下结构
"""

## 输入

你会收到一个 JSON，包含：
- link_verdict：Step 2 的链路判断结论（定性名字、链路形态、风险等级等）
- cluster_data：三个簇的数据，每簇有 gujia_uv/xiadan_uv/chengjiao_orders/chengjiao_gmv + 所属品类列表
- last_week_strategies：上周策略清单文本

## 你必须执行的判断

### 1. 上周策略验证（必须先做）

对每个簇，检查上周策略是否兑现：
- 兑现 → "策略有效，可继续" + 给出 KPI 数据
- 未兑现 → "策略无效" + 给出原因（外部变了/策略错了/执行问题）
- 无法验证 → "数据不足，继续观察"

### 2. 流量端 vs 转化端分开归因

对每簇：
- 流量端信号 = gujia_uv.qoq（估价UV 环比）
- 转化端信号 = chengjiao_orders.qoq 减去 gujia_uv.qoq
  - 差值 > 0 → 转化在改善
  - 差值 < -3pp → 转化在恶化
  - 差值在 [-3, 0] 区间 → 转化平稳

### 3. 归因二维矩阵（v0.4，责任主体 × 时间属性）

**不再用"行情/动作/双重/待查"单标签。** 拆两个正交维度，各带证据：

**维度一 责任主体**（MECE 五选，可多个，每个都要证据）：
- 市场行情：公开行情信号（显卡涨价/新品/以旧换新），看同簇是否同步动
- 我方动作：本周运营动作（活动/补贴/push/提价），看动作覆盖品类是否同步动
- 竞对动作：友商顶价/降价/抢量，需竞品数据
- 用户结构变化：高/低客单用户占比变化
- 不可归因：三查后仍无信号（**必须真查过，说明排除了哪些，不是偷懒逃生舱**）

**维度二 时间属性**（MECE 三选一）：
- 一次性事件：大促/单笔大单/政策窗口，不可复现
- 短期波动：1-2周可回归
- 结构性趋势：连续3周同向 + 同比印证（对齐 Step 2 趋势标签）

**输出**：`责任主体`（数组）+ `时间属性`（单值）+ `证据`（每个主体对应证据）。除"不可归因"外每个主体必须给证据。

### 3.5 品类间此消彼长（v0.4，品类下钻时判断）

对每个下跌品类，判断流量去向：
- 流失量 ≈ 同簇某涨品类的增量，总量不变 → `内部竞争(流向X品类)`，调分配而非救单品
- 没有对应的涨，整簇缩量 → `外部流失`，按责任主体归因
- 流量流向其他簇 → `跨簇转移(流向Y簇)`，需上报做结构性判断

### 3.8 论点交叉仲裁（v0.4）

若本簇归因结果与 Step 2 的大盘定性标签冲突（如 Step2 定性"量退价补"=被动，本簇归因=我方动作=主动）：
- 我方动作是局部(仅此簇) → 保留 Step 2 大盘定性，标注"本簇为主动逆势"
- 我方动作是全局(影响大盘) → 记入 `仲裁`，建议修正塔尖标签
- 冲突无法解决 → 标注"存疑并列"

发生仲裁时输出到 `仲裁` 数组：`{原判, 证据, 修正为}`。

### 4. 资源分配信号

对每簇输出：
- 影响度：占大盘 GMV 比例
- 可解决度：等市场恢复 / 自己可控可改 / 不可改
- 建议操作：继续投入 / 暂缓观察 / 放弃 / 下钻

### 5. 品类下钻

对异常品类，给：
- 异常品类名 + 流量/转化判断
- 影响度%（占簇 GMV 比例）
- 是否值得下钻到机型层

值得下钻的条件（满足任一）：
- 影响度 > 20%
- 环比异常并连续 2 周同向
- 是重点品类（组装机/运动相机/无人机/白酒/显卡/内存条/台球杆）
