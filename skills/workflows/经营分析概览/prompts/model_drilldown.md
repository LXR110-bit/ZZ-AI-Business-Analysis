## 前置条件

你仅会在 Step 3 判断"值得下钻到机型层"时才被调用。当前数据一定是合适的。

## 输入

你会收到一个 JSON，包含：
- 品类名：当前分析的品类名
- models：机型列表，每项有 model_name/gujia_uv/chengjiao_orders/转化率

## 你必须执行的判断

### 1. 机型四象限定位

计算品类均值：
- 中位估价UV = 所有机型 gujia_uv 的中位数
- 中位转化率 = 所有机型转化率的中位数

对每机型：

| 场景 | 判断 | 优先级 |
|---|---|---|
| 估价 UV > 中位 + 转化率 < 中位 | P0 定价问题，优先提价 | 🔴 |
| 估价 UV > 中位 + 转化率 > 中位 | P2 健康机型，保持 | 🟢 |
| 估价 UV < 中位 + 转化率 > 中位 | P1 精准用户，扩流量 | 🟡 |
| 估价 UV < 中位 + 转化率 < 中位 | P3 边缘机型，不处理 | - |

### 2. 提炼机型规律

该品类有没有可复用的机型规律？

已知规律：
- 无人机：新品发布 → 带动上一代旗舰 + 高持有量入门款是主力回收
- 运动相机：Pocket 新品 → 同生态机型增长

### 3. 跨品类复用

- 有品牌关联（大疆/苹果/索尼等）→ 可复用至同品牌其他品类
- 有品类关联（手机→平板→穿戴）→ 可复用至关联品类
- 都没有 → 仅限本品类
