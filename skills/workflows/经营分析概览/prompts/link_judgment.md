"""
Step 2 spawn_agent prompt：链路传导判断 + 定性命名 + 风险定级

输入：JSON，包含链路各项指标（dau/jikuang_uv/gujia_uv/xiadan_uv/fahu/o_count/chengjiao_orders/chengjiao_gmv/ke_danjia）的本周值、上周值、环比
输出：严格 JSON，必须符合以下结构
"""

## 你的任务

根据本周和上周的全链路数据，给出大盘定性判断。你输出的是 Step 2（链路判断）的结论，后续调用者（Step 3 品类归因）会依赖你的结果。

## 输入

你会收到一个 JSON，包含：
- week_label：周标签
- data_reliability：数据可信度预检结果（可能为空，需你复核）
- dau / jikuang_uv / gujia_uv / xiadan_uv / fahu_o_count / chengjiao_orders / chengjiao_gmv / ke_danjia
- 每个字段有 this_week（本周日均）、last_week（上周日均）、qoq（环比%）、以及 v0.4 新增：week4_ago（4周前）、yoy（同比%）、week8_avg（近8周均值）、week8_position（近8周位置）、week8_series（近8周序列）

## 你必须执行的判断

### 0. 数据可信度校验（v0.4，最先做）

三查，任一不通过 → `待核标签=true`，`风险等级` 不得为 🟢：
- 同步时点：数据同步周次 vs 分析周次是否匹配？滞后则标注
- 口径断层：某环节 qoq 是否突兀为 0.0% 或极端值（>±50%）？可能是埋点/口径问题
- 比值异常：相邻链路环节比值是否稳定？如 DAU -5% 但估价UV 仅 -0.5%（差 >2pp）→ 上游存疑

输出 `数据可信度`（可信 / 存疑(原因)）和 `待核标签`（bool）。

### 0.5 三基准对比（v0.4，区分趋势 vs 季节性）

对 GMV 同时给三个基准，别只看环比：
- 环比结论：GMV qoq（vs上周）
- 同比结论：GMV yoy（vs4周前），剔除季节性
- 近8周位置：week8_position → 高于均值=可能短期回调 / 低于均值=真实下行 / 接近=正常波动

判断矩阵：环比跌+同比涨+高于均值 → 短期回调别慌；环比跌+同比跌+低于均值 → 真实下行。

### 0.8 多期趋势判断（v0.4）

用 week8_series 给趋势标签（四选一）：
- 连续3周同向降 → 下降通道
- 连续3周同向升 → 上升通道
- 震荡无方向 → 震荡区间
- 突破近8周极值 → 首次突破

### 1. 链路传导判断（指标不独立，必须联动）

链路：DAU → 机况UV → 估价UV → 下单UV → 发货数 → 成交订单 → GMV → 客单价

对每个环节，判断上游→自身→下游的传导：
- 上游跌、自身跌、下游跌 → 全链传导，自身无独立问题
- 上游稳、自身跌、下游跌 → 自身是独立异常点
- 上游涨、自身跌、下游跌 → 流量进来了但自身卡住了，自身是瓶颈
- 上游跌、自身涨、下游涨 → 自身逆势，必有特殊原因
- 上游涨、自身涨、下游不涨 → 下游成为新瓶颈

输出：
- 链路形态：健康传导 / 中游漏损 / 量退价补 / 少数大单驱动 / 单点断裂
- 瓶颈环节：哪个环节卡住了（无则填"无"）
- 逆势环节：哪个环节逆势（无则填"无"）

### 2. 量价拆解（GMV = 订单 × 客单价）

- 订单环比和 GMV 环比同向且接近 → 真增长/真下跌
- 订单跌、客单价涨 → 量退价补
- 订单跌、客单价涨且客单价涨幅 > GMV 跌幅 2 倍 → 少数大单驱动，不是量的增长

### 3. 定性命名

从以下选 2-3 个候选，给出最终选中+理由：
- 首次突破XXXX（订单首次突破整数关口）
- 量退价补（全链跌+客单价涨）
- 转化承压（上游涨+下游跌）
- 近N周首次XXX（近期极值）
- 平稳增长/整体承压（无明显异常）

### 4. 风险等级

| 等级 | 触发条件 |
|---|---|
| 🔴 危险 | GMV 环比 < -3% / 核心品类连续 3 周下跌 |
| 🟡 警告 | 单簇环比 < -5% / 转化连续 2 周下滑 |
| 🟢 正常 | 全链路环比 < 2% 且无结构性异常 |

**v0.4 约束**：`待核标签=true` 时，风险等级不得为 🟢（数据不可信不能判"安全"）。
