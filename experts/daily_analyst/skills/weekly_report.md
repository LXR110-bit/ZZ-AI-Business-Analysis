# 周报 Skill

## 一、输入识别

从用户问题里提取：
- **品类**：iPhone、iPad、Mac、安卓、配件 ……
- **周期**：本周（默认）、上周、指定 ISO 周（如 2026-W26）
- **粒度**：品类整体 / 子型号 / 渠道
- **特殊要求**：是否要对比同期？是否要分渠道？

> 不明确就反问，**不要瞎猜**。

## 二、数据获取

1. **查邮件**：调 `read_email` 找匹配的数据邮件
   - 主题规则：`[Agent数据][周报][<品类>][<周期>]`
   - 例：`[Agent数据][周报][iPhone][2026-W26]`
2. **解析 CSV**：调 `parse_csv`
3. **校准口径**：每个指标都用 `query_metric` 确认定义（特别是 GMV / 转化率 / 客单价 / 复购率口径）

## 三、必含模块（每篇周报固定结构）

按以下 5 个模块顺序输出，**缺一不可**：

### 模块 1：核心指标（最多 5 个）
- 成交单量 / 成交 GMV / 客单价 / 转化率 / 复购率
- 每个指标都要 ✅ 同比 + ✅ 环比 + ✅ 在生命周期里的位置（§2）

### 模块 2：异动追踪（最关键）
对每个变化幅度 > 5% 的指标，强制走 **§4 异动诊断四问**：
1. 数据可信吗？
2. 短期还是结构性？
3. 外部还是内部？
4. 对应什么动作？

### 模块 3：三层穿透（§1）
本周必须明确回答：
- **上游侧**变化？（新品、上游货源、竞品供给）
- **市场侧**变化？（爱回收/闲鱼/竞品价格走势、用户认知）
- **我们侧**变化？（活动、转化、定价、产品改版）

### 模块 4：瓶颈定位（§3）
- 列出完整链条：曝光 → 进店 → 加购 → 估价 → 下单 → 成交 → 复购
- 标注本周瓶颈环节（哪个转化率最低？拐点最早？）
- 围绕瓶颈给建议

### 模块 5：下周动作建议（§5 必须闭环）
- 不超过 3 条
- 每条**必须**带：验证方法 / 基线 / 预期增量 / 成本 / ROI
- 没法闭环的建议（如 "加强用户运营"）禁止输出

## 四、输出格式（写入飞书文档）

```markdown
# 【品类周报】<品类> · <周期> · <ISO 周>

## 一句话结论
（金字塔顶层，控制在 30 字以内，可让运营立刻知道好不好）

## 核心指标
| 指标 | 本期 | 同比 | 环比 | 阶段判断 |
|---|---|---|---|---|
| ... | | | | |

## 异动追踪
### <指标 1> ±X%
- 可信性：...
- 持续性：...
- 来源：...
- 对应动作：...

## 三层穿透
- **上游**：...
- **市场**：...
- **我们**：...

## 瓶颈定位
当前瓶颈：<环节>，转化率 <X%>（vs 基线 <Y%>）
证据：...

## 下周动作（不超过 3 条）
| # | 动作 | 验证 | 基线 | 预期 | 成本 | ROI |
|---|---|---|---|---|---|---|
| 1 | ... | ... | ... | ... | ... | ... |

## 附录
- 数据来源：邮件 `[Agent数据][周报][...]`
- 取数 SQL ID（如有）：...
- 数据范围：<起止时间>
```

## 五、自检清单（§6 强制）

输出前 Agent 自查：

- [ ] 一句话结论是不是真"一句话"？（≤ 30 字）
- [ ] 5 个核心指标都带同比 + 环比 + 阶段？
- [ ] 异动条目都走完 §4 四问？
- [ ] §1 三层全覆盖（不要漏一层）？
- [ ] 瓶颈定位有数据证据？
- [ ] 3 条动作建议都有 §5 完整闭环？
- [ ] 没有"技术口径"未翻译？
- [ ] 没有未来源的数字？

**任一项 NO → 回炉**。

## 六、输出后操作

1. 调 `write_doc` 在飞书"品类周报"文件夹下创建文档
2. 调 `send_im` 把文档链接推送给请求方
3. 如运营有追问，转移到对话模式（不要在周报里回答追问）
