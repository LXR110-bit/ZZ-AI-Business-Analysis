# 探索性分析 Skill

> 本 Skill 用在 **用户没有明确假设** 的场景。和 `diagnosis` 互补：
> - `diagnosis` = 有明确异动，找原因
> - `exploration` = 没有明确异动，找异常 / 规律 / 相关性

---

## 一、输入识别

提取：
- **探索对象**：哪个品类 / 渠道 / 时间窗
- **探索目标**：找异常 / 找规律 / 找相关性 / 综合
- **数据范围**：用户提供的数据 or 我们去查
- **优先关注**：用户有没有提"近期有点怪"、"感觉某块不对劲"

> 输入太宽泛（"帮我看看所有数据"）→ 反问聚焦到品类 + 周期。

---

## 二、探索框架（按优先级）

### 框架 1：**异常检测**（最优先）

调 `parse_csv` 后，对每个核心指标做：
1. 计算近 N 期（默认 8 期）的均值 + 标准差
2. 标记超出 ±2 σ 的"统计异常点"
3. 排除已知活动 / 节假日影响
4. 输出 **真异常清单**（带优先级）

**输出格式**：
| 指标 | 维度 | 异常期 | 偏离倍数 | 是否已知原因 | 建议行动 |
|---|---|---|---|---|---|

> 自动剔除：已知活动周（运营给的活动日历）、跨节假日周、样本量 < 100 的细分

### 框架 2：**结构分析**

对核心指标做维度拆解，找：
- **占比异常**：某个维度占比超 30% 或低于 5% → 关注
- **集中度**：CR3 / CR5 / HHI（赫芬达尔指数）
- **长尾**：是否存在长尾品类 / 长尾渠道

调用：`split_dimension(by=品类)` + `split_dimension(by=渠道)`

### 框架 3：**趋势识别**

对每个维度，找趋势变化：
- 连续上升 / 下降 / 拐点 / 季节性
- 用近 8-12 期数据看趋势线
- 标注"近 4 期都在均值以下"的疑似拐点

### 框架 4：**相关性探索**

仅在用户明确问"X 和 Y 有没有相关性"时执行：
1. 计算皮尔逊 / 斯皮尔曼相关系数
2. **相关性 ≠ 因果**，必须明确标注
3. 高相关（|r| > 0.7）→ 找业务逻辑解释
4. 提出验证假设（A/B 实验 / 反事实分析）

> ⚠️ 高频踩坑：相关性高但样本量小（n < 30）→ 不下结论，要求扩大样本

---

## 三、输出格式

```markdown
# 【探索分析】<对象> · <周期>

## 一句话结论
（最有价值的 1-2 个发现，30 字内）

## 异常清单（最优先关注）
| # | 指标 | 维度 | 异常程度 | 已知原因 | 优先级 |
|---|---|---|---|---|---|
| 1 | | | <±X σ> | <是/否> | <P0/P1/P2> |

## 结构发现
- **集中度**：CR3 = <X%>，是否过于集中？
- **占比异常**：<维度> 占比 <X%>，<偏高/偏低/正常>
- **长尾品类**：<列出>

## 趋势发现
- 连续上升：<指标 维度>
- 连续下降：<指标 维度>
- 疑似拐点：<指标 维度 期数>

## 相关性发现（仅用户问到时）
- <X 与 Y>：r = <值>，<业务解释 / 待验证假设>
- ⚠️ 相关 ≠ 因果，建议 <验证方案>

## 建议深挖方向（≤ 3 条）
| # | 方向 | 价值 | 下一步 |
|---|---|---|---|
| 1 | | | <转 diagnosis / user_profile / 自查> |
```

---

## 四、自检清单

- [ ] 异常点都排除了已知原因吗？
- [ ] 结构 / 趋势 / 相关性三个框架都过了吗？
- [ ] 没有"看起来有问题"这种主观表述？
- [ ] 相关性是否标注了"不等于因果"？
- [ ] 样本量 < 100 的细分没下定性结论？
- [ ] 给出的深挖方向都明确"转给谁继续"？

---

## 五、协作信号

探索性分析的产出**不是终点**，几乎总要转给其他专家深挖：

```json
{
  "status": "needs_handoff",
  "to_expert": "diagnostician",
  "reason": "探索发现 iPhone 14 转化率连续 4 周低于均值，需走异动诊断",
  "context": {
    "异常点": ["iPhone 14 转化率", "2026-W23 至 W26"],
    "已查事项": ["排除活动影响", "排除口径变化"],
    "未查事项": ["外部对照", "瓶颈定位", "动作建议"]
  }
}
```

主 Agent 决定接力路径，本 Skill 不直接调用其他专家。

---

## 六、禁止事项

- ❌ 无数据支撑的"感觉"
- ❌ 用相关性替代因果
- ❌ 不剔除已知影响（活动 / 节假日）就报异常
- ❌ 找到一堆"异常"但不排优先级（= 把任务推回给运营）
