# 用户画像 Skill

> 用本 Skill 回答 **"是谁"** 类问题。画像不是目的，是后续运营动作（人群圈选 / 触达 / 投放）的输入。

---

## 一、输入识别

提取：
- **画像对象**：成交用户 / 估价用户 / 流失用户 / 某品类用户 / 某活动参与用户
- **画像维度**：年龄 / 性别 / 地域 / 消费力 / 设备 / 渠道来源 / 品类偏好 / 转转身份（C2C 卖家 / B2C 买家）
- **对比对象**：vs 大盘 / vs 同品类历史 / vs 竞品（若有外部数据）
- **应用目的**：圈人触达 / 投放定向 / 产品优化 / 还是单纯认知

> 没有应用目的的画像 = 浪费分析。**反问**用户后续要做什么，再决定画哪几个维度。

---

## 二、画像生产流程

### 步骤 1：取数 + 校口径

1. `read_email` 取画像源数据（一般是 BI 跑好的明细 CSV）
2. `parse_csv` 解析
3. `query_metric` 校准每个维度的口径定义（特别是 "活跃用户" / "成交用户" 这种容易歧义的）

### 步骤 2：基础分布

调 `split_dimension` 对每个维度拆解：
- 年龄段：18-24 / 25-30 / 31-40 / 41-50 / 50+
- 性别：男 / 女 / 未知
- 地域：一线 / 新一线 / 二线 / 三四线 / 五线及以下
- 消费力：高 / 中 / 低（按平台等级或历史 GMV 分位）
- 渠道：自然 / 信息流 / 私域 / 短信 / push
- 设备：iOS / Android / 其他

> 维度过多反而冲淡价值。**默认输出 3-5 个最关键维度**，多余的折叠到附录。

### 步骤 3：差异化分析（最关键）

只看分布意义不大，要找 **"vs 对照组" 的显著差异**：

| 维度 | 本组占比 | 对照组占比 | 差异（pp） | 是否显著 |
|---|---|---|---|---|

**显著性判定**：
- 差异 > 5 pp 且本组样本 > 100 → 显著
- 差异 > 10 pp → 强显著，必须解读
- 样本量 < 100 → 不做差异判断（§7 兜底）

### 步骤 4：业务解读（不解读 = 浪费）

每个显著差异必须回答：
- **为什么是这群人？**（业务逻辑解释）
- **能不能用？**（是否可圈人触达 / 投放）
- **怎么用？**（具体动作建议）

## 三、输出格式

```markdown
# 【用户画像】<画像对象> · <周期>

## 一句话画像
（30 字内，例："成交用户主要是 25-35 岁一线城市男性，iOS 占 70%，自然流量为主"）

## 核心维度分布
| 维度 | 占比最高的 3 组 | 备注 |
|---|---|---|
| 年龄 | 25-30 (35%) / 31-40 (28%) / 18-24 (20%) | |
| 性别 | 男 70% / 女 28% | |
| 地域 | 一线 40% / 新一线 25% / 二线 18% | |

## 差异化发现（vs 大盘）
| 维度 | 本组 | 大盘 | 差异 | 业务解读 |
|---|---|---|---|---|
| | | | | |

## 可应用动作
| # | 圈人逻辑 | 触达方式 | 预计样本 | 预期效果 |
|---|---|---|---|---|
| 1 | <如：iOS 一线城市 30-40 岁男性> | <短信/push/广告投放> | <X 万人> | <按 §5 闭环填> |

## 数据来源
- 邮件：`[Agent数据][画像][...]`
- 样本量：<N 人>
- 取数 SQL ID：<id>
```

## 四、自检清单

- [ ] 画像对象 vs 对照组都明确了吗？
- [ ] 每个维度的样本量都 > 100 吗？小样本组标"不下结论"了吗？
- [ ] 差异化发现都给了业务解读吗（不是纯数字）？
- [ ] 给了至少 1 条可应用动作吗？
- [ ] 动作建议有 §5 闭环（验证 / 基线 / 预期 / 成本 / ROI）吗？
- [ ] 一句话画像真的是 30 字内？

## 五、协作信号

画像结论常作为 **diagnostician** 或 **daily_analyst** 的输入：

```json
{
  "status": "needs_handoff",
  "to_expert": "diagnostician",
  "reason": "画像发现新增用户中三四线占比 +15 pp，需诊断是否因为某个渠道变化",
  "context": {
    "画像结论": "新增用户三四线占比从 28% → 43%",
    "时间窗": "2026-W24 至 W26",
    "已查事项": ["排除节假日影响"]
  }
}
```

---

## 六、禁止事项

- ❌ "用户主要是年轻人" 这种无量化的描述
- ❌ 画完图就完事、不给应用动作
- ❌ 用全量画像替代差异化分析（占比 30% 男性 = 大盘也是 50% 男性，等于没分析）
- ❌ 小样本（< 100）下定性结论
- ❌ 涉及性别 / 地域 / 年龄等敏感属性时缺乏业务依据
