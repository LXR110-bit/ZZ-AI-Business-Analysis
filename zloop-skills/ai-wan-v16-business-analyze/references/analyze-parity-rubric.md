# AI小万 analyze 阶段 · GPT-5.5→Claude 平价 Rubric

> 目的：让 Claude Sonnet 的 analyze 输出与 GPT-5.5 金标（W27/W28/W29）**结论等价、结构一致、证据纪律一致**。
> 用法：① 作为 analyze prompt 的硬约束内联；② 作为 Review Gate 的评分项；③ 作为 golden eval 的 diff 依据。
> 判定原则：不追字面一致，追「同样识别异动品类 / 同样归因方向 / 同样行动建议 / 同样带数纪律 / 同样 schema」。

---

## 0. 全局纪律（每条输出都必须满足，违反即 FAIL）

1. **每句带数**：任何定性判断后必须跟客观数字（GMV日均X万、环比±X万、降幅X%、某率±X个百分点）。
2. **零主观词**：禁止「效果显著 / 明显改善 / 大幅提升 / 表现优异」作为结论。量级词（大幅/明显/偏大）**只有紧跟具体数字时才允许**（如「估价完成率下降3.51个百分点、降幅偏大」可），单独作结论禁止。
3. **量价四象限**：大盘/分层必须给量价判断，受控词：`量价同降 / 量跌价升 / 量降价升 / 量价同升`。
4. **链路统一口径**：链路拆解固定走 `机况UV→估价UV→估价完成率→下单率→发货率→成交率`。
5. **异常必标口径**：低基数/异常波动品类必须写「先核查口径 / 跨周归因差异」，不直接给经营动作。
6. **计划只指链路段**：计划里只写要查哪一段链路（如「先查估价UV到下单」），**不臆测具体业务机制**（用户取消率/质检标准/价格波动等），除非 evidence_pack 有支撑。
7. **假设只在高层级**：`board / tiers` 必须落「预期验证（假设）」；**品类条不逐条甩假设**，只对头部高影响或异常品类补一句，其余品类到「先查X→再查Y」即可（对齐金标紧凑度）。

---

## 1. 输出 schema（字段与金标一致）

```
insights:
  board                 : str   # 大盘
  category              : str   # 全局品类概览
  tiers                 : {发展, 孵化, 种子}  # 每层一段
  secondaryCategories   : {板块名: str}       # 13 板块
  categories            : {品类名: str}        # 全部品类逐条
  monitor               : str   # 监测总提示
```

---

## 2. 各字段 rubric（金标 DNA）

### board（大盘）
- [ ] 风险等级（偏高/中等/低）
- [ ] 量价判断（四象限词）
- [ ] 核心指标带数：GMV日均+环比±+降幅、成交订单±、客单价±
- [ ] 链路指标带数：机况UV/估价UV/估价完成率/下单率/发货率/成交率 各变化
- [ ] 拖累源点名（发展层 + 具体品类）+ 机会点名
- [ ] 收尾给下钻方向（三段链路：估价→下单 / 下单→发货 / 发货→成交）

### category（全局品类概览）
- [ ] 大盘下滑/上涨归因到分层
- [ ] **优先下钻对象点名**（这就是 Phase 1 交接单的 always_floor 来源）
- [ ] 机会点名 + 「需区分真实增长 vs 低基数波动」

### tiers（发展/孵化/种子，每层）
- [ ] 品类数 + GMV日均 + 环比± + 降幅/增幅
- [ ] 成交订单± + 四个率各变化（估价完成率/下单率/发货率/成交率，单位「个百分点」）
- [ ] 核心问题（点名拖累品类）
- [ ] 机会（点名增长品类）
- [ ] 计划：先查X→再查Y + 预期验证（假设句）

### secondaryCategories（板块，13 条）
- [ ] 贡献点名 + 拖累点名
- [ ] 波动特征（头部贡献大但某某承压）
- [ ] 下钻建议（点名品类 + 链路段）

### categories（品类逐条，全部）
- [ ] **定性标签**（受控词，见 §3；按 §3.1 判定矩阵选，不凭感觉）
- [ ] GMV日均 + 环比±/降幅
- [ ] 链路指标带数（哪个 UV/率 升降几个百分点）
- [ ] 计划：先查X→再查Y（链路段）
- [ ] **紧凑**：单条对齐金标长度（约 60–110 字）；假设句只对头部高影响或异常品类补，普通品类不甩假设
- [ ] **空品类唯一化**：GMV=0 / 无成交的品类不能都套同一句模板（会触发全局重复率≥20% 阻断）；每条至少带品类名 + 一句差异化说明（如"本周无有效成交，估价UV/机况UV 具体数字，建议核口径"）

### monitor（监测）
- [ ] 总提示 + 异常品类点名（口径稳定性）
- [ ] 旧版写「不输出机型级AI分析」；**新版两跳设计中此处由 Phase 2 机型归因补充**

## 3. 受控词表（controlled vocabulary，降低跨模型方差的关键）

- 品类定性标签：`高影响风险品类` / `明确机会品类` / `异常风险品类` / `低基数波动品类` / `稳健品类`
- 量价判断：`量价同降` / `量跌价升` / `量降价升` / `量价同升`
- 风险等级：`偏高` / `中等` / `低`
- 链路段：`机况UV→估价UV` / `估价UV→下单` / `下单→发货` / `发货→成交`

> 强制模型从受控词表选词，是让两个不同模型「读起来一致」最省力的一招。

### 3.1 品类定性标签判定矩阵（消除标签主观性，周周一致的关键）

标签 = 影响度 × 方向，按顺序判定（命中即停，第 5 条兜底保证每个品类都有标签）：

| 优先级 | 标签 | 判定条件 |
|---|---|---|
| 1 | `异常风险品类` | 指标出现**不合常理**的值（如成交率单周变化数十/上百个百分点、成交率 > 100%、低基数品类成交订单远高于流量），只能用口径/归因差异解释 → 必须先核查口径。注意：头部品类某漏斗率真实下滑几个百分点属真经营问题，归 5「高影响风险」而非本条 |
| 2 | `低基数波动品类` | GMV 日均 < 5 万（低量级），涨跌大概率小样本放大 → 只做数据/链路核验 |
| 3 | `明确机会品类` | GMV 环比正增长 且下单率/成交率等链路指标同向改善 |
| 4 | `稳健品类` | GMV 降幅 < 3% 且无链路指标明显恶化（各率变化在 ±0.3 个百分点内） |
| 5 | `高影响风险品类` | **兜底**：以上未命中的 GMV 下滑品类（即有意义的下滑且非低基数）。GMV 日均越大越优先下钻；影响度由文案里的 GMV 数字体现 |

> 阈值（5万低量级 / 3% 降幅 / 0.3 个百分点 / 单率3个百分点畸变）是默认口径；若 evidence_pack/server_context 给了分层或基线阈值，以后者为准。判定用到的量级/降幅数字必须写进文案，做到「标签可由数字复核」。矩阵 exhaustive：任何品类都能落到唯一标签。

## 4. Eval 判定（golden diff）

对每周，用本 rubric 对 Claude 输出打分，并与同周 GPT-5.5 金标 diff：

| 维度 | 通过标准 |
|---|---|
| 异动品类命中 | Claude 点名的优先下钻品类 ⊇ 金标 always_floor 集（发展层头部） |
| 归因方向 | 量价判断 + 链路断点方向与金标一致 |
| 行动建议 | 每个风险项有「先查X→再查Y」；board/tier 有预期验证假设，品类条不逐条甩假设 |
| schema 合规 | 6 段齐全、字段名一致 |
| 带数纪律 | 抽查 10 句，全部带数字、无主观词 |
| 受控词 | 定性标签/量价词全部落在 §3 词表内 |

达标线：命中 + 方向 + schema + 带数 四项必须全过；行动 + 受控词允许 1 处 warn。
